# JavLibrary

**本地影视库管理系统**

由于jellyfin和Emby，以及主流NAS的播放应用对某些小众影片的支持不太友好，所以借助AI自己动手做了一个。

基于 Electron + Vue 3 构建的桌面应用，自动扫描和解析本地 NFO 元数据文件，提供多维度浏览、搜索和管理功能（需搭配其它元数据刮削工具使用）。

**版本**: v2.8.1

## 功能特点

- 🌓 **主题切换**: 日间/夜间模式切换，页面、组件与弹窗随主题联动（果然影视墙还是黑底更适合啊！）
- 📁 **自动扫描**: 自动扫描本地文件夹中的 NFO 元数据文件，构建本地数据库
- ⚙️ **多路径支持**: 支持添加多个数据文件夹路径，数据叠加管理
- 🔍 **多维度浏览**: 支持按演员、导演、制作商、分类、系列等多维度浏览影片
- 🔎 **高级搜索**: 提供简易搜索和多重搜索两种模式，支持多条件组合查询
- 🎯 **列表页筛选器**: 所有影片列表页顶部支持按分类标签筛选
- 🎰 **随机摸奖**: 选择困难症犯了？试试随机抽奖吧！
- ❤️ **收藏夹**: 图文列表页、影片详情页可点击收藏，支持多收藏夹
- 🧩 **分类管理**: 分类 Tab 支持编辑当前分类列表与一键导入默认分类
- 🔄 **数据同步**: 启动时与磁盘 diff 自动增删库；支持「完整扫描」与「仅扫描新增或修改」
- ✏️ **数据编辑**: 支持编辑影片信息，仅更新 NFO 中修改的字段、保留其余内容（建议使用前备份 NFO，确认符合预期后再批量使用）
- 🎬 **视频播放**: 播放器开发不在本项目的考虑范围内，为调用系统默认播放器播放

## 技术栈

- **Electron**: 跨平台桌面应用框架
- **Vue 3**: 前端框架
- **Vite**: 快速构建工具
- **SQLite**: 轻量级本地数据库
- **Sequelize**: Node.js ORM 框架
- **Element Plus**: Vue 3 UI 组件库

## 开发环境要求

- Node.js 18+
- npm 或 pnpm

## 安装依赖

```bash
# 安装根目录依赖
npm install

# 安装前端依赖
cd frontend
npm install
cd ..
```

## 开发运行

```bash
# 同时启动 Electron 和前端开发服务器
npm run dev

# 或者分别启动
npm run dev:electron  # 启动 Electron
npm run dev:frontend  # 启动前端开发服务器（端口 5173）
```

## 构建打包

```bash
# 构建前端
npm run build:frontend

# 打包 Electron 应用
npm run build:electron

# 快速构建（跳过 sqlite3 重新编译）
npm run build:quick
```

## 使用说明

### 首次启动

1. 启动应用后，系统会提示选择 data 文件夹路径
2. 选择包含 NFO 文件的文件夹
3. 在设置页面点击"开始扫描数据文件夹"进行初始数据导入

### 数据格式

data 文件夹结构示例：
```
data/
├── 文件夹目录1/
│   ├── 作品文件夹1/
│   │   ├── poster.jpg    # 封面（或 *ps.* / 文件名含 poster，如 ABC-001-poster.jpg）
│   │   ├── fanart.jpg    # 海报（或 *pl.* / 文件名含 fanart，如 ABC-001-fanart.jpg）
│   │   ├── xxx.nfo       # 元数据文件（任意 .nfo 后缀均可）
│   │   ├── video.mp4     # 视频文件（可选）
│   │   └── extrafanart/  # 预览图文件夹（可选，详情页多图轮播；文件夹名含 extrafanart 即可）
│   │       ├── 1.jpg
│   │       └── ...
│   └── ...
└── ...
```

- **NFO**：支持任意文件名的 `.nfo` 文件，不限于 `movie.nfo`
- **封面/海报**：优先识别以 `ps`/`pl` 结尾的图片；其次按关键词匹配，文件名含 `poster` 为封面、含 `fanart` 为海报（支持 `poster.jpg`、`标题-poster.jpg` 等）
- **预览图**：作品文件夹下名称含 `extrafanart` 的子文件夹内图片会作为详情页预览图，与详情主图一起可点击轮播放大
- **识别码**：NFO 中可使用 `<uniqueid>` 或 `<num>` 标签

### 如何准备演员数据（可选）

若需在应用中显示演员头像，可拉取 [gfriends/gfriends](https://github.com/gfriends/gfriends) 项目；在设置中选择的演员数据路径**必须包含** `Filetree.json` 与 `Content` 目录。选好路径后点击「扫描演员信息」即可。
若需自建或修改数据，请参阅 [演员头像数据格式说明（Filetree.json + Content）](演员头像数据格式说明.md)。

### 主要功能

- **影片详情**: 点击影片进入详情页，可查看完整信息、编辑数据、播放视频
- **收藏夹管理**: 图文列表页及详情页可对影片进行收藏，支持自定义收藏夹
- **目录浏览**: 在"目录"页面通过下拉框切换不同类型的目录（演员、文件目录、导演、制作商、分类）；演员目录下支持放大/缩小按钮调整网格卡片大小
- **搜索功能**: 在"搜索"页面使用简易搜索或多重搜索查找影片；简易搜索支持输入即搜（防抖 0.7 秒），结果在页内展示
- **设置管理**: 在"设置"页面配置影片/演员数据路径、扫描数据、调整显示和同步设置
- **演员头像**: 可选配置演员数据路径后，演员列表页（仅女优模式）、详情页、目录页展示头像；未配置时演员列表页不显示头部区域，配置后无头像时显示占位保持布局；支持简繁体与曾用名匹配，多头像时可选择默认展示
- **演员名称与曾用名**: 演员列表页（女优模式）名称旁可编辑显示名称与曾用名（仅改应用库，不修改 NFO）；曾用名参与「扫描演员信息」时的头像匹配

### 配置说明

应用开箱即用，如需自定义配置：

- **影片数据路径**: 在设置页面添加或删除数据文件夹路径（支持多路径，必填）
- **演员数据路径**: 可选。选择包含 `Filetree.json` 与 `Content` 目录的文件夹，用于演员头像展示；设置后需点击「扫描演员信息」构建映射（支持简繁体匹配）
- **自动同步**: 开启"启动时自动扫描"可在应用启动时监听文件变化并自动同步数据
- **显示设置**: 开启"仅显示可播放"可过滤掉没有视频文件的影片
